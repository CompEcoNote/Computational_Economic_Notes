# Submission guidelines


Authors submit either by:


1. Forking the `paper-template` repository, filling it, and opening a Pull Request to the journal's `submissions` collection (advanced), **or**
2. Creating a new Issue using the submission template and linking to their paper repository.


## Required metadata


- Title
- Authors and affiliations
- Abstract (max 300 words)
- Keywords
- License (recommended CC-BY 4.0)


## File formats


- Preferred: Markdown + references.bib (build pipeline will create PDF)
- LaTeX is also supported. See the `paper-template` folder in this repo.


## Peer review process


- Submissions are triaged by editors.
- Selected reviewers are invited and asked to comment on the submission Issue.
- Reviews and author responses are tracked publicly in the Issue thread.


## Acceptance and publication


- Accepted papers are published on the website under `/papers/YYYY/` and a DOI can be minted via Zenodo.
